# 手動テストチェックリスト v0.3

v0.2 のチェックリスト（`manual-test-checklist-v0.2.md`）をベースに、v0.3 HUD演出強化の確認項目を追加したもの。
基本フロー（動画選択・セッション・設定保存・結果画面）の項目は v0.2 のチェックリストを併用する。

## v0.3: HUD視認性

- [ ] 下部応援メッセージが読みやすい（背景グラデーションで動画と重なっても判読できる）
- [ ] 応援メッセージが2行以内に収まり、自然に折り返す
- [ ] 中央演出が長すぎない（CLEAR/FEVER/LAST PUSHは約1秒、FINISHのみ少し長い）
- [ ] CLEAR / FEVER / LAST PUSH / FINISH が重なりすぎない（高優先度が勝つ）
- [ ] 動画終了直前に FINISH が表示される
- [ ] high density（多め）でもメッセージが速すぎず読み切れる

## v0.3: レイアウト

- [ ] 縦向きでsafe-areaが問題ない（上部HUDがノッチに食い込まない・下部がホームインジケータに隠れない）
- [ ] 結果画面のボタンが画面下に隠れない
- [ ] 横向きにすると「縦向きでの利用がおすすめです」バナーが出る
- [ ] バナーの×で案内を閉じられる
- [ ] 小画面幅（〜360px）でトーン・頻度チップUIが破綻しない
- [ ] 画面高さが低い環境でHUDが詰まりすぎない

## v0.3: アクセシビリティ・演出抑制

- [ ] prefers-reduced-motion で中央演出がフェードのみになる
- [ ] prefers-reduced-motion でゲージ・メッセージのアニメーションが抑えられる
- [ ] トーン・頻度チップがキーボード（Tab + Enter）で操作できる

## v0.3: デバッグモード

- [ ] `?debugHud=1` でHUD確認ができる（動画なし）
- [ ] `?debugHud=1&phase=push&callout=fever&density=high` で状態指定できる
- [ ] debugHud設定がlocalStorageに保存されない
- [ ] debugHudなしの通常利用に影響がない
