# v0.3 HUD演出強化 計画

v0.3-alpha（feature/v0.3-hud-polish）で第一段階を実装済み。

## 実装済み（v0.3-alpha）

- HUDレイヤー整理（`hudLayers.ts` でz-index一元管理、top/side/bottom/center/banner/debug）
- 中央演出の短時間化と優先度制御（`calloutQueue.ts`、FINISH > LAST PUSH > FEVER > CLEAR）
- FINISHカットイン追加（動画終了直前）
- 下部応援メッセージの背景改善（グラデーション・clamp()・max-width・line-height）
- 縦画面UIのsafe-area調整（結果画面・下部メッセージ・低画面高さ）
- prefers-reduced-motion時の演出抑制（フェードのみ・transition無効）
- 小画面端末向けフォントサイズ調整（is-small-widthクラス + clamp）
- landscape時の案内表示（OrientationBanner）
- HUDデバッグモード（`?debugHud=1&phase=&callout=&tone=&density=`）
- スクリーンショット用のサンプル状態（デバッグモードで再現可能）

## v0.3正式版前に確認すること

- iPhone Safari 実機（safe-area・HUD視認性・完走フロー）
- Android Chrome 実機（同上）
- 実動画でのFINISHカットイン（PC Chromeでは実測済み。実機で再確認）
- prefers-reduced-motion のOS設定での実挙動（ロジック/CSSはテスト済み）
- safe-area実機（ノッチ・ホームインジケータ）
- 横向き案内の実機での回転時挙動
- メッセージ可読性の実機での見え方
- debugHudが通常利用へ影響しないこと（ローカルでは確認済み）

## v0.3正式版で微調整すること

- セッションゲージのフェーズ連動の色変化
- 端末差の吸収（フォント・余白の微調整）
- version を 0.3.0 に上げ、タグ・Release を作成

## v0.4以降に送ること

- サバイバルモード有効化（DANGERゲージ・LAST STAND・SURVIVED演出）
- PWA化

## 目的

- HUD演出を気持ちよくする
- 動画中央の邪魔を減らす
- prefers-reduced-motion 対応を強化する
- safe-area と画面サイズ別調整を改善する
- CLEAR / FEVER / LAST PUSH / FINISH の表示タイミングを磨く
- 応援メッセージの読みやすさを上げる

## やる候補

| 項目 | 概要 | 主な対象ファイル |
|---|---|---|
| HUDレイヤー整理 | z-index / グリッドの整理と命名統一 | `Hud.tsx` / `Hud.css` |
| 中央演出の短時間化 | カットイン表示時間・アニメ曲線の調整 | `CenterCallout.tsx` |
| 下部応援メッセージの背景改善 | 動画と重なった時の可読性（グラデーション・ぼかし） | `CheerMessage.tsx` / `Hud.css` |
| セッションゲージの視認性改善 | 太さ・配色・フェーズ連動の色変化 | `SessionGauge.tsx` |
| 縦画面UIのsafe-area調整 | ノッチ・ホームバー端末での余白最適化 | `global.css` / `Hud.css` |
| reduced-motion時の演出抑制 | カットインをフェードのみへ、パルス無効化 | 各CSS |
| 小画面端末向けフォントサイズ調整 | 360px幅以下での clamp 調整 | `tokens.css` ほか |
| landscape時の案内表示 | 横向き時「縦向きでの利用を推奨」の表示 | `App.tsx` / 新規CSS |
| HUDデバッグモード | 動画なしでHUD状態を再現するdevモード（?debug=hud） | 新規 `hooks` / `App.tsx` |
| スクリーンショット用サンプル状態 | README/ストア用の見栄え状態を再現 | デバッグモードに含める |
